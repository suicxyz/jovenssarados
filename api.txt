A API sempre retornará um JSON com o parâmetro "status", que pode ter dois valores, "ERROR" e "OK".
Quando o valor de "status" for igual a "ERROR", existirá sempre um parâmetro "message", com os possíveis valores:

ANA - Action Non-Authorized   	- Ação não autorizada, redirecionar usuário para página de login.
DNF - Document Not Found      	- Registro não encontrado.
DAE - Document Already Exist  	- Já existe um registro com a mesma credencial (email, cpf, telefone etc).
FNV - Format Not Valid        	- Formato inválido (CPF, email, etc).
ONC - Operation Not Concluded 	- Operação não concluída, erro desconhecido.
INS - ID Not Specified        	- ID não especificado

Quando o valor de "status" for igual a "OK", o parâmetro "message" existirá nos seguintes casos:

DD - Document Deleted			- Registro excluído
AA - Access Authorized 		- Acesso Autorizado
